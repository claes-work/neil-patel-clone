# Why You Should NEVER Redesign Your Website, Do THIS Instead

## Summary
Neil argues against full website redesigns, especially for a business already up and running and making money. His core point is conversion: even if a new design looks better and loads faster, there's no guarantee it converts as well as the old one, and if it converts worse you lose money. The overlooked factor in redesigns is conversion optimization. Instead of a wholesale redesign, tweak the site **element by element, page by page**, and A/B test each change (he names Optimizely and Crazy Egg) to see which version drives more revenue. If the old "ugly" version drives more revenue, keep it; if the new one wins, switch and slowly roll it out across the site. The A/B test is the safeguard that ensures a design decision doesn't set the business back.
