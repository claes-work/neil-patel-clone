# How to Speed Up Your Wordpress Website (in just 5 steps)

## Summary
A five-step WordPress site-speed playbook opened with a Portent study stat on
conversion loss per second of load time. Steps: (1) check site speed via
neilpatel.com/seo-analyzer (an Ubersuggest-family tool) and target under three seconds,
diagnosing common slowdown causes (bad hosting config, no page caching, unoptimized
image file size, poorly-coded/excessive plugins, too many external scripts); (2)
install a caching plugin (recommends the free WP Super Cache, claims 2-5x load-time
improvement); (3) optimize images/video for mobile (teases an unreleased Ubersuggest
WordPress image-optimization plugin, offering early access via comments); (4) use a CDN
(explains via his own site being served from a geographically local server, e.g. from
India, names Cloudflare); (5) use lazy loading, citing a first-party result of ~6%
bounce-rate reduction in low-connectivity regions after implementing it.

## Key claims
- First-party data disclosure: implementing lazy loading reduced bounce rates by
