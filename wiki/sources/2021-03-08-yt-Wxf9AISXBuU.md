# Here's How You Assess and Improve a Website's SEO - Neil Patel's Website SEO Breakdown Session

## Summary
A live site-breakdown webinar co-hosted with **Saulo Medeiros** (CMO at UX, described
as the event host/moderator; Brazilian audience). Neil live-audits several
audience-submitted sites using a **five-bucket scoring framework** (Content/search
intent, Off-page, On-page, Technical, Mobile), each scored 0–10, with the philosophy
that fixing the lowest-scoring bucket first beats trying to fix everything at once.
Content bucket: pursue transactional-intent keywords rather than raw volume; the
"2,000 words to rank" rule is a myth — word count should match search intent, not a
fixed target; keyword density of 1–3% is a red flag for unnatural stuffing, not a
target to hit; recommends 3–4 blog posts per weekday as an ideal cadence (states his
own team publishes ~70 pieces of content/month toward an 80–100/month goal, using
revision/rewrite staff so he doesn't write everything himself). On-page bucket: one
H1 tag per page (like a book title), correct heading hierarchy (H2 sections, H3
subsections), keyword in title/first paragraph/URL, short URLs preferred, image file
names and alt text should be descriptive, 3–4 blog posts/posts-per-week internal
linking guideline of 5–15 links per 1,000 words. Off-page bucket: cites a stat that
82% of consumers check reviews/comments before buying; backlink growth should be
slow and steady — sudden referring-domain spikes are a red flag worth investigating
(not necessarily manipulation, but worth checking). Technical/Mobile buckets:
recommends Ubersuggest's Site Audit tool for automated technical/Core Web Vitals
checks. CRO: cites a specific A/B test on neilpatel.com/consulting where moving a
lead-gen form field from the right side to the left side of the page produced a
**54% increase in conversions** with identical copy.
