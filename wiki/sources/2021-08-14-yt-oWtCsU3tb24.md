# NOT Every Keyword is Worth Pursuing in SEO (Here's Why)

## Summary
Argues against chasing high-competition/high-volume keywords on low-authority sites,
recommending easier wins first and reinvesting into harder terms later. Names
**Google's RankBrain** algorithm update explicitly as the shift from keyword-matching
to intent-detection (the "apple" computer-vs-fruit disambiguation example), reinforcing
the existing "it's about serving user intent, not keyword-stuffing" belief with a named
algorithm citation not previously logged this explicitly. Repeats the <0.1%-of-9M-
visitors conversion self-critique. Walks the standard Ubersuggest workflow: Keyword
Ideas report → "Keywords I Can Rank For" (logged-in, URL-filtered) → prioritize by SEO
difficulty score → Site Audit report (priority-ordered fixes) → Rank Tracking project
setup to monitor mobile/desktop rankings across cities/countries over time.

## Key claims
- Names Google's **RankBrain** algorithm update as the mechanism behind the shift from
  keyword-matching to serving user intent, using the "apple" (computer vs. fruit)
  disambiguation example (paraphrase, dated 2021-08) — seo. First explicit RankBrain
  citation logged in this run; reinforces the existing intent-over-keyword-stuffing
  belief with a named technical reference.
- Recommends prioritizing keywords by Ubersuggest's SEO difficulty score over raw
  relevance/volume when starting out, to generate faster wins that fund longer-term
  investment in harder terms (paraphrase, dated 2021-08) — seo. Reinforces existing
  quick-wins-first doctrine.
